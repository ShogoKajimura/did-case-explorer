Governance and operations

Designed for long-term maintenance, not a one-off demo.

The public explorer is static by design, but the restricted OCR intake and private status workflow already run on a separate backend.

Recommended host

Cloudflare Pages is the preferred deployment target for the public explorer because it serves static assets globally, honors security headers, and leaves room for later Workers/R2-based private workflows.

Security posture

Keep the public build free of contributor uploads and restricted PDFs. The browser should only receive metadata, coded outputs, version manifests, and links that are already legally public.

Maintenance model

Treat each corpus freeze as a versioned data snapshot. Publish a new build only when metadata, coding, or legal-access status has been audited and signed off.

Deferred to phase 2

Upload intake, private OCR return artifacts, contributor accounts, and any restricted storage should remain outside this public app until a separate authenticated backend is designed.
